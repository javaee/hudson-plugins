For multi-node cluster, GlassFish DAS and the first GlassFish Instance is deployed on current computer (build executor).

Rest of the Instances are deployed on other Hudson nodes - called GlassFish plugin subslaves.

Logic:

  • Number Of Instance: i

  • Number Of Nodes: n

  1. If n = 1 : Create Single Node Cluster - Start i instances on 1 Node.

  2. If n >= i : Create Multi Node Cluster, Single Instance Per Nodes

  3. If 1 < n < i : Start i / n instances on each node, in a round robin fashion.

For Example, if the user Selects:

  • Number Of Instance: 7

  • Number Of Nodes: 3

Then instance Names will be: in1, in2, in3, in4, in5, in6, in7

Node1 will run: DAS, in1, in4, in7

Node2 will run: in2, in5

Node3 will run: in3, in6
